
# سياسة الإصدارات (Versioning Policy)

يتبع نظام **Cash & Custody** منهجية الإصدارات التالية لضمان الاستقرار:

### 1. تصنيف الإصدارات
- **Patch (v1.0.1):** إصلاح أخطاء برمجية (Bugs) أو أمنية حرجة. لا يوجد تغيير في الواجهة.
- **Minor / Beta Update (v1.1.0-beta.2):** تحسينات في تجربة المستخدم (UX)، نصوص الواجهة، أو إضافة فهارس أداء.
- **Major (v2.0.0):** تغيير في دورة العمل (Workflow)، هيكل قاعدة البيانات (Schema)، أو إضافة ميزات ضخمة.

### 2. شروط إصدار Beta.2
سيتم الانتقال للنسخة التجريبية الثانية عند تجمع 5 ملاحظات تحسينية (UX Improvements) أو الحاجة لدعم ميزة "التقارير المخصصة".

### 3. التجميد (Freeze)
خلال فترة Beta.1، يمنع منعاً باتاً تعديل الـ RPCs أو الـ RLS Policies لضمان سلامة بيانات العميل الأول.
